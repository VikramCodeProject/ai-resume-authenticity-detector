# Render.com Deployment Guide

This guide walks you through deploying the Resume Verification System on Render.com.

## Prerequisites

1. **Render Account**: Create a free account at [https://render.com](https://render.com)
2. **GitHub Repository**: Push your code to GitHub (public or private)
3. **Environment Variables**: Prepare all required secrets

## Step 1: Create GitHub Connection

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **"New+"** → **"Blueprint"**
3. Connect your GitHub account (authorize Render)
4. Select the repository containing this project

## Step 2: Configure render.yaml

The `render.yaml` file in the root directory defines all services:

- **PostgreSQL Database** - Persistent data storage
- **Redis Cache** - Session and caching layer
- **Backend (FastAPI)** - Python API server
- **Frontend (React)** - Web UI

No additional configuration needed unless you want to change:
- **Region**: Modify `region: oregon` to your closest region (us-west, us-east, eu-west, etc.)
- **Plan tier**: Change `plan: starter` to `standard` for production workloads

## Step 3: Deploy Using Blueprint

1. In Render Dashboard, click **"New+" → "Blueprint"**
2. Select your GitHub repository
3. Choose the branch (usually `main` or `master`)
4. The `render.yaml` will be automatically detected
5. Click **"Create New Blueprint"**

## Step 4: Set Environment Variables

After services are created, configure the following in Render Dashboard:

### Backend Environment Variables

Go to **Backend Service → Environment** and add (or sync from `.env.production`):

```
ENVIRONMENT=production
JWT_SECRET=<generate-a-32+-char-random-string>
GITHUB_API_KEY=<your-github-token>
LINKEDIN_API_KEY=<your-linkedin-token>
AWS_S3_BUCKET=<your-s3-bucket-name>
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=<your-aws-access-key>
AWS_SECRET_ACCESS_KEY=<your-aws-secret-key>
ETH_RPC_URL=https://polygon-rpc.com
SMART_CONTRACT_ADDRESS=<your-deployed-contract-address>
PRIVATE_KEY=<your-wallet-private-key>
DATABASE_URL=<automatically-set-by-Render>
REDIS_URL=<automatically-set-by-Render>
FRONTEND_URL=<automatically-populated-with-frontend-url>
```

### PostgreSQL Environment Variables

Go to **Database Service → Environment** and verify:
```
POSTGRES_USER=postgres
POSTGRES_PASSWORD=<auto-generated-strong-password>
POSTGRES_DB=resume_verify
```

### Frontend Environment Variables

Go to **Frontend Service → Environment** and add:
```
VITE_API_URL=https://<your-backend-service-name>.onrender.com
NODE_ENV=production
```

## Step 5: Configure Database Connection

### Initial Setup

1. Wait for PostgreSQL service to be healthy (it will show a green checkmark)
2. Get the database connection string from **Database Service → Info**
3. Run migrations (if using Alembic):
   ```bash
   # From your local machine or via Render shell
   alembic upgrade head
   ```

### Alternative: Using Render's Database

To use Render's managed PostgreSQL instead of Docker:
1. Create a new PostgreSQL service in Render Dashboard
2. Copy the internal connection string
3. Paste it in Backend service's `DATABASE_URL` variable
4. Delete the Docker PostgreSQL from `render.yaml`

## Step 6: Handle File Uploads

Since Render's file system is ephemeral (resets on deploy), configure AWS S3:

1. Create an S3 bucket on AWS
2. Set IAM user with S3 access
3. Add to environment variables:
   - `AWS_S3_BUCKET`
   - `AWS_ACCESS_KEY_ID`
   - `AWS_SECRET_ACCESS_KEY`
4. Update backend code to use S3 for resume storage

**Important**: Do NOT store files locally on Render!

## Step 7: Blockchain Configuration (Optional)

For blockchain features:

1. Get a wallet private key (use a development/test wallet)
2. Deploy the smart contract to Polygon (testnet or mainnet)
3. Set environment variables:
   - `SMART_CONTRACT_ADDRESS` - Contract address after deployment
   - `PRIVATE_KEY` - Wallet private key (use Render secrets!)
   - `ETH_RPC_URL` - Already set to Polygon RPC

**Security**: Always use separate wallets for production!

## Step 8: Custom Domain (Optional)

1. In Render Dashboard, go to **Frontend Service → Settings**
2. Click **"Add Custom Domain"**
3. Follow domain provider instructions for DNS setup

## Step 9: SSL/TLS Certificate

Render automatically provides SSL certificates for:
- `yourservice.onrender.com` domains
- Custom domains (auto-generated by Render)

No additional setup needed!

## Step 10: Monitoring & Logs

### View Logs

1. **Backend**: Dashboard → Backend Service → Logs
2. **Frontend**: Dashboard → Frontend Service → Logs
3. **Database**: Dashboard → Database Service → Logs
4. **Redis**: Dashboard → Redis Service → Logs

### Health Checks

Each service includes health checks:
- Backend: `/api/health`
- Frontend: `/`
- Database: PostgreSQL ping
- Redis: PING command

Render will auto-restart services if they fail health checks.

## Deployment Stages

1. **Building** (5-10 min)
   - Services start building
   - Dependencies are installed
   - Docker images are created

2. **Deploying** (2-5 min)
   - Services are deployed
   - Environment variables are set
   - Databases are initialized

3. **Running** 
   - All services are healthy
   - Application is live and accessible

## Troubleshooting

### Backend won't start
- Check logs: `Dashboard → Backend → Logs`
- Common issues:
  - Missing environment variables
  - Database not accessible
  - Port already in use
  - Missing dependencies in `requirements.txt`

**Fix**: Add missing variables or dependencies, then click **"Manual Deploy"**

### Frontend build fails
- Check logs: `Dashboard → Frontend → Logs`
- Common issues:
  - Missing npm dependencies
  - TypeScript compilation errors
  - Incorrect `VITE_API_URL`

**Fix**: Update code or environment, commit, Render auto-deploys

### Database connection timeout
- Ensure PostgreSQL is healthy (green status)
- Check firewall rules (should be automatic on Render)
- Verify `DATABASE_URL` matches service

### Redis connection issues
- Check Redis service is running (green status)
- Verify `REDIS_URL` in environment
- Check if services are in same region

### Slow startup
- Increase service plan tier
- Optimize dependencies in `requirements.txt`
- Use gunicorn workers: `--workers 4` (configurable in `render.yaml`)

## Update & Redeployment

### Auto Deployment

If `autoDeploy: true` in `render.yaml`:
- Render auto-deploys on Git push
- Takes 5-10 minutes
- Old services remain until new ones are healthy

### Manual Redeployment

1. Dashboard → Service → **"Manual Deploy"**
2. Render rebuilds and redeploys
3. No downtime with blue-green deployment

## Security Checklist

- [ ] Database password is strong (auto-generated by Render)
- [ ] JWT_SECRET is at least 32 characters
- [ ] AWS credentials have minimal IAM permissions (S3 only)
- [ ] Private key uses test/development wallet (not mainnet)
- [ ] CORS is configured to frontend domain only
- [ ] Database is NOT publicly accessible
- [ ] All secrets are in Render (not in git history)
- [ ] SSL/TLS is enabled (automatic)
