# Render.com Deployment Checklist

## Pre-Deployment (Local Setup)

- [ ] Project code is clean and working locally
- [ ] All tests pass: `pytest tests/`
- [ ] No hardcoded secrets in code or config files
- [ ] `.gitignore` includes `.env`, `.env.*.local`, etc.
- [ ] Git repository is up to date
- [ ] GitHub remote is configured: `git remote -v`

## GitHub Setup

- [ ] Repository is pushed to GitHub (public or private)
- [ ] Main/master branch is current
- [ ] No sensitive data in commit history
- [ ] `.gitignore` is properly configured
- [ ] GitHub access token created (if using private repo)

## Render Account Setup

- [ ] Render account created at https://render.com
- [ ] GitHub account is connected to Render
- [ ] GitHub repository is authorized for Render
- [ ] Render dashboard is accessible

## Configuration Files

- [ ] `render.yaml` exists in root directory
- [ ] `backend/Dockerfile` configured (if using Docker)
- [ ] `frontend/Dockerfile` configured (if using Docker)
- [ ] `.env.render.production` template created
- [ ] `RENDER_DEPLOYMENT_GUIDE.md` is available

## Environment Variables - Backend

Required variables to set in Render Dashboard → Backend Service → Environment:

### Security
- [ ] `JWT_SECRET` - 32+ character random string (generate: `openssl rand -hex 32`)
- [ ] `ENVIRONMENT=production`

### Database (Auto-populated by Render)
- [ ] `DATABASE_URL` - From PostgreSQL service
- [ ] `REDIS_URL` - From Redis service

### External APIs
- [ ] `GITHUB_API_KEY` - GitHub personal access token
- [ ] `LINKEDIN_API_KEY` - LinkedIn API key

### AWS S3 (File Storage)
- [ ] `AWS_S3_BUCKET` - S3 bucket name created
- [ ] `AWS_REGION` - AWS region (default: us-east-1)
- [ ] `AWS_ACCESS_KEY_ID` - IAM access key
- [ ] `AWS_SECRET_ACCESS_KEY` - IAM secret key

### Blockchain (Optional)
- [ ] `ETH_RPC_URL` - Polygon RPC endpoint
- [ ] `SMART_CONTRACT_ADDRESS` - Deployed contract address
- [ ] `PRIVATE_KEY` - Wallet private key (use test wallet)

### Frontend Integration
- [ ] `FRONTEND_URL` - Auto-populated from Frontend service

## Environment Variables - Frontend

- [ ] `VITE_API_URL` - Backend service URL (auto-populated)
- [ ] `NODE_ENV=production`

## External Services Setup (Before Deploy)

### AWS S3
- [ ] AWS account created
- [ ] S3 bucket created
- [ ] IAM user created with S3 permissions
- [ ] Access key created
- [ ] Bucket region noted

### GitHub
- [ ] Personal access token created
- [ ] Token has appropriate permissions
- [ ] Token expiration set (1 year recommended)

### Blockchain (Optional)
- [ ] Smart contract deployed to Polygon/Ethereum
- [ ] Contract address saved
- [ ] Wallet created (test wallet for development)
- [ ] Private key backed up securely

## Deployment Steps

1. [ ] Push code to GitHub: `git push origin main`
2. [ ] Go to https://dashboard.render.com
3. [ ] Click "New+" → "Blueprint"
4. [ ] Connect GitHub and select repository
5. [ ] Select branch (usually `main`)
6. [ ] Review `render.yaml` (will be auto-detected)
7. [ ] Click "Create Blueprint" or "Deploy"
8. [ ] Wait for services to build and deploy (5-15 minutes)

## Troubleshooting

If services fail to deploy:

1. **Check logs**: Dashboard → Service → Logs
2. **Common issues**:
   - Missing environment variables
   - Incorrect database connection string
   - Port conflicts
   - Missing system dependencies
3. **Recovery**: Fix issue → Commit → Git push → Render auto-redeploys
